Glosario

A

  • Accesibilidad web: Práctica de diseñar y desarrollar sitios y aplicaciones web que puedan ser utilizados por todas las personas, incluidas aquellas con discapacidades. Esto implica garantizar que el contenido sea perceptible, operable, comprensible y robusto para usuarios con diferentes capacidades y tecnologías de asistencia.
  • Add: Comando que se utiliza para agregar cambios en los archivos al staging area, preparándolos para ser incluidos en el siguiente commit.
  • Atributos: Valores adicionales que se agregan a las etiquetas para proporcionar información o personalizar su comportamiento. Se escriben dentro de la etiqueta de apertura y suelen tener un formato de clave-valor.

C

  • Cascada: Mecanismo que determina cómo se aplican los estilos cuando hay reglas conflictivas. Si múltiples reglas afectan a un mismo elemento, la cascada decide cuál aplicar según la especificidad del selector, el orden de aparición en el código y la importancia de las reglas.
  • Chromium project: Proyecto de navegador web de código abierto iniciado por Google, que sirve como la base para varios navegadores modernos como Google Chrome, Microsoft Edge, Opera, Brave, etc.
  • Commit: Registro o "foto" de los cambios actuales realizados en los archivos de un repositorio. Incluye un mensaje descriptivo que explica los cambios y crea un punto en el historial del proyecto al que se puede regresar en el futuro.
  • CSS: Se conoce como "Hojas de estilo en cascada", y es un lenguaje utilizado para definir el estilo y la presentación de una página web. Permite controlar aspectos como el diseño, los colores, las fuentes y la disposición de los elementos HTML en una página, mejorando su apariencia visual.

D

  • DevTools: Conjunto de herramientas integradas en un navegador web, que permiten a los desarrolladores inspeccionar, depurar y optimizar el código de una página web en tiempo real. Incluyen funcionalidades para ver el HTML, CSS y JavaScript, analizar el rendimiento, depurar errores y probar diseños responsivos.
  • Display: Propiedad que define cómo se muestra un elemento en una página web. Los valores más comunes son block (el elemento ocupa todo el ancho disponible y comienza en una nueva línea), inline (el elemento solo ocupa su contenido y no fuerza un salto de línea) e incline-block (combina las características de inline y block).

E

  • Emmet: Herramienta que permite escribir código HTML y CSS de forma más rápida y eficiente mediante abreviaciones, llamadas snippets, que se expanden en estructuras completas.
  • Especificidad: Manera en la que se determina qué regla de estilo se aplica cuando hay múltiples reglas que afectan a un mismo elemento. Se calcula en función de la cantidad y tipo de selectores utilizados, como etiquetas, clases, ID y atributos. Los selectores con mayor especificidad tienen prioridad sobre los de menor especificidad.
  • Etiquetas: Son los elementos básicos en HTML que definen la estructura y el contenido de una página web. Se escriben entre corchetes angulares (< >), que indican al navegador cómo interpretar y mostrar el contenido de una página web. Cada etiqueta define un elemento, como texto, imágenes, enlaces o secciones, organizando y estructurando el documento.
  • Extensiones: Son complementos que se pueden instalar en VS Code para agregar nuevas funcionalidades al editor, como soporte para lenguajes de programación, herramientas de depuración, temas de color, integración con Git y otras utilidades.

F

  • Fork: Copia de un repositorio que permite hacer cambios de forma independiente al repositorio original. Se utiliza para proponer mejoras o trabajar en un proyecto sin afectar el repositorio principal. Después de realizar cambios, es posible crear un pull request.
  • Formularios: Elemento que permite recuperar información del usuario y enviar dicha información a un servidor para que pueda ser procesada.
  • Fusión: También conocido como merge, es el proceso de combinar los cambios de una rama con otra. Esto se utiliza comúnmente para integrar el trabajo de una rama de desarrollo o funcionalidad a la rama principal del proyecto, manteniendo un historial consolidado.

G

  • Git: Sistema de control de versiones distribuido que permite gestionar y realizar un seguimiento de los cambios en los archivos de un proyecto.
  • GitHub: Servicio basado en la nube que aloja un sistema de control de versiones llamado Git. Éste permite a los desarrolladores colaborar y realizar cambios en proyectos compartidos, a la vez que mantienen un seguimiento detallado de su progreso.
  • GitHub Pages: Servicio de alojamiento de sitio estático que toma archivos HTML, CSS y JavaScript directamente desde un repositorio en GitHub, opcionalmente ejecuta los archivos a través de un proceso de complilación y publica un sitio web.

H

  • Herencia: Mecanismo por el cual ciertos estilos aplicados a un elemento se transmiten a sus elementos hijos. Por ejemplo, propiedades como el color y la fuente de un texto se heredan automáticamente por los elementos dentro de un contenedor, a menos que se especifique lo contrario.
  • HTML: Lenguaje de marcado utilizado para crear y estructurar contenido en la web. Utiliza etiquetas (como div, img, p, a) para definir elementos como textos, imágenes, enlaces y otros componentes de una página web.

I

  • IDE: Aplicación de software que ayuda a los programadores a desarrollar código de software de manera eficiente. Reúne tod lo necesario para escribir, probar y depurar código.
  • Inteligencia artificial: Campo de la informática que se enfoca en crear sistemas y programas capaces de realizar tareas que normalmente requieren inteligencia humana.

M

  • Mini-sprint: Variación del sprint tradicional en metodologías ágiles, adaptada para intervalos de tiempo más cortos, cuyo objetivo es permitir a los equipos trabajar en tareas específicas con un enfoque intensivo y obtener resultados rápidamente.
  • Modelo de caja: Describe cómo se estructura el espacio de un elemento, incluyendo el contenido, el relleno, el borde y el margen. Cada parte afecta el tamaño y la disposición del elemento en la página.

N

  • Navegador web: Aplicación que permite acceder, visualizar e interactuar con sitios y aplicaciones web en Internet, como por ejemplo Mozilla Firefox, Google Chrome, Microsoft Edge, etc.
  • Node.js: Entorno de ejecución de JavaScript del lado del servidor que permite ejecutar código JavaScript fuera del navegador, lo que facilita la creación de aplicaciones web y servidores altamente escalables y eficientes.

P

  • Posicionamiento: Propiedad que permite controlar cómo se coloca un elemento en una página web. Puede ser static, relative, absolute, fixed o sticky, cada uno controlando su ubicación de diferentes maneras respecto al flujo del documento o la ventana del navegador.
  • Prompt: Instrucción que se proporciona a un modelo de lenguaje para generar una respuesta o completar una tarea. Estos pueden variar, desde simples frases hasta preguntas complejas y comandos detallados.
  • Propiedades: Características que se pueden aplicar a los elementos HTML para definir su apariencia o comportamiento, desde el color y el tamaño hasta el diseño y la animación. Cada propiedad tiene un valor que especifica cómo debe mostrarse el elemento.
  • Pseudo-clases: Palabras clave que se utilizan para definir estilos a un elemento, en función de su estado o condición especial.
  • Pseudo-elementos: Palabras clave que permiten aplicar estilos a partes específicas de un elemento, como el primer carácter o la primera línea de un texto, o incluso crear contenido visualmente antes o después de un elemento.
  • Pull: Comando que se emplea para extraer y descargar contenido desde un repositorio remoto y actualizar al instante el repositorio local para reflejar ese contenido.
  • Pull request: Petición que el propietario de un fork de un repositorio hace al propietario del repositorio original para que este último incorpore los commits que están en el fork.
  • Push: Comando que se utiliza para enviar los cambios locales (commits) de un repositorio a un repositorio remoto. Esto actualiza el repositorio remoto con los cambios que se han realizado localmente, permitiendo que otros colaboradores accedan a ellos.

R

  • Rama: Son líneas independientes de desarrollo dentro de un repositorio. Permiten trabajar en diferentes funcionalidades, correcciones o experimentos sin afectar la rama principal (main o master).
  • README: Archivo de tipo Markdown (.md) que proporciona información sobre el proyecto de un repositorio. Suele incluir detalles como la descripción del proyecto, cómo instalarlo, cómo usarlo, y cualquier otra información relevante para los usuarios o colaboradores. Este archivo se visualiza automáticamente en la página principal del repositorio.
  • Repositorio: Contenedor donde se almacena todo el historial de cambios de un proyecto, incluidos los archivos, las ramas y las versiones. Puede ser local (en tu máquina) o remoto (en plataformas como GitHub) y sirve para gestionar y colaborar en el desarrollo de proyectos de manera eficiente.

S

  • Selectores: Son patrones utilizados para seleccionar y aplicar estilos a elementos HTML específicos. Pueden basarse en el tipo de etiqueta, clase, ID, atributos o relaciones entre los elementos.
  • Semántica: Se refiere al uso de etiquetas que describen claramente el propósito y el significado del contenido que contienen. Por ejemplo, etiquetas como header, article o footer indican la función de esas secciones en lugar de solo definir su apariencia, mejorando la accesibilidad y el SEO.
  • Sprint: Período de tiempo corto y definido, generalmente de una a cuatro semanas, durante el cual un equipo se enfoca en completar una cantidad específica de trabajo.
  • Staging area: Área intermedia donde se colocan los cambios que se desean incluir en un próximo commit. Permite preparar y revisar qué modificaciones se registrarán, brindando control sobre qué partes del proyecto se guardan en el historial.

T

  • Terminal: Interfaz de línea de comandos, mediante la cual es posible interactuar directamente con el sistema operativo haciendo uso de comandos de texto.

V

  • Visual Studio Code: Editor de código fuente desarrollado por Microsoft. Ofrece herramientas para escribir, depurar y ejecutar código en varios lenguajes de programación, con soporte para extensiones y personalización.
